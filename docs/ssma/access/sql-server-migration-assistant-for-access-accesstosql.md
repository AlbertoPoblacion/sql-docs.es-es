# <a name="sql-server-migration-assistant-for-access-accesstosql"></a>SQL Server Migration Assistant para Access (AccessToSQL)

[!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Migration Assistant (SSMA) para el acceso es una herramienta para migrar bases de datos de las versiones de acceso [!INCLUDE[msCoName](../../includes/msconame_md.md)] 97 a 2010 a [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 2012, [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 2014, [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 2016, [!INCLUDE[msCoName](../../includes/msconame_md.md)] @ no__t-9 2017 en Windows y Linux, 0 @ No__ t-11 2019 en Windows y Linux, o 2 Azure SQL Database. SSMA para Access convierte los objetos de base de datos de Access en objetos [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] o Azure SQL Database, carga esos objetos en [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] o Azure SQL Database y, a continuación, migra los datos de Access a [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] o Azure SQL Database.
  
En esta documentación se ofrece una introducción a SSMA para el acceso y se proporcionan instrucciones paso a paso para migrar bases de datos de Access a [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] o Azure SQL Database e información sobre los problemas que pueden producirse después de la migración.
