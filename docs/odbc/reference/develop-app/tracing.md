# <a name="tracing"></a>Seguimiento
El Administrador de controladores ODBC tiene un recurso de seguimiento que permite que la secuencia de llamadas de función realizadas por una aplicación ODBC se grabe y transcriba en un archivo de registro. El seguimiento se realiza mediante un archivo DLL de seguimiento que captura las llamadas entre la aplicación y el Administrador de controladores y entre el Administrador de controladores y el controlador. Este método de seguimiento reemplaza el seguimiento realizado por el Administrador de controladores ODBC 2 *.x* y el seguimiento realizado en ODBC 2 *.x* por ODBC Spy.
