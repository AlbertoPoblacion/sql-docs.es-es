# <a name="sql-data-developer"></a>Desarrollador de datos SQL
Utilice la plataforma de datos SQL de Microsoft para crear soluciones centradas en datos en dispositivos móviles y equipos de escritorio para servidores Web, servidores empresariales y la nube.  

## <a name="sql-data-storage"></a>Almacenamiento de datos SQL
* [SQL Server motor de base de datos](../database-engine/configure-windows/sql-server-database-engine.md): utilice el motor de base de datos SQL Server para crear bases de datos relacionales para el procesamiento de transacciones en línea o datos de procesamiento analítico en línea. 
* [SQL Azure](https://docs.microsoft.com/azure/sql-database/): usar SQL Azure para trasladar la base de datos a la nube 
* [SQL Compact](https://www.microsoft.com/download/details.aspx?id=17876): SQL Server Compact para compilar aplicaciones independientes y en ocasiones conectadas para dispositivos móviles, escritorios y clientes Web.
* [LocalDB](../database-engine/configure-windows/sql-server-2016-express-localdb.md): Use LocalDB durante el desarrollo para escribir y probar el código de TRANSACT-SQL sin tener que administrar una instancia de servidor completa de SQL Server.

## <a name="sql-data-tools"></a>SQL Data Tools
* [SQL Server Data Tools](../ssdt/download-sql-server-data-tools-ssdt.md) : Use SSDT para crear bases de datos relacionales, bases de datos SQL de Azure, paquetes de Integration Services, modelos de datos de Analysis Services e informes de Reporting Services.
* [Herramientas de administración de SQL Server](../ssms/download-sql-server-management-studio-ssms.md): Use SSMS configure, supervise y administre instancias de SQL Server.

## <a name="sql-data-access"></a>Acceso a datos de SQL
* [Controladores de cliente SQL](sql-connection-libraries.md): Use controladores SQL para conectarse, consultar, actualizar, insertar o eliminar datos de una base de datos SQL.
* [Entity Framework](https://msdn.microsoft.com/library/gg696172.aspx): Use el Entity Framework para tener acceso fácilmente a las bases de datos mediante LINQ, con acceso directo a SQL Server y la opción de controlar la asignación entre una Entity Data Model (EDM) y las estructuras relacionales sin procesar. 
* [Windows Communication Foundation (WCF)](https://msdn.microsoft.com/library/dd456779.aspx): Use Data Services para una solución casi llave en mano con el fin de crear y consumir fácilmente servicios de oData en la web y en la intranet.
* [Sync Framework](https://msdn.microsoft.com/library/jj839436.aspx): Use el marco de sincronización para habilitar el acceso sin conexión a cualquier tipo de datos, almacén de datos, cualquier protocolo de transferencia y cualquier topología de red.
* [Extensiones](https://msdn.microsoft.com/library/hh242985.aspx)reactivas: Use las extensiones reactivas (RX) para realizar la programación de secuencias de eventos y simplificar la programación asincrónica mediante secuencias observables y operadores de consulta de estilo Linq.  Use las extensiones reactivas para JavaScript (RxJS) para simplificar la programación asincrónica basada en la devolución de llamada y la programación controlada por eventos en la web mediante secuencias observables.
* [Integración CLR](../relational-databases/clr-integration/common-language-runtime-clr-integration-programming-concepts.md): use la Integración CLR para escribir procedimientos almacenados, desencadenadores, tipos definidos por el usuario, funciones definidas por el usuario, agregados definidos por el usuario y funciones con valores de tabla de transmisión de datos usando cualquier lenguaje de .NET Framework, incluidos Microsoft Visual Basic .NET y Microsoft Visual C#. 
* [Sqlxml 4,0](../relational-databases/sqlxml/sqlxml-4-0-programming-concepts.md): usar SQLXML 4,0 para exportar datos relacionales a XML.
